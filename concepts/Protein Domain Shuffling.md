# Protein Domain Shuffling

## Definition
Evolutionary process where modular protein domains are recombined into new arrangements, creating novel multi-domain architectures. Same domains, different orders and combinations → new functions. The "Lego blocks" of protein evolution.

## Mechanism: Exon Shuffling
- **Domains** correspond to exons in genomic DNA
- **Intron-mediated recombination** enables clean domain swapping
- Recombination at intron-exon boundaries preserves domain integrity
- Non-homologous recombination creates new domain combinations

## Domains as Building Blocks
Approximately **10,000 domain families** (Pfam database) combine to create millions of proteins. Most eukaryotic proteins are multi-domain.

### Example Domain Architectures
- **SH2 domain** + **Kinase domain** → Src family kinases
- **Immunoglobulin domain** × N repeats → Antibodies, cell adhesion molecules
- **DNA-binding domain** + **Transactivation domain** → Transcription factors
- **Protease domain** + **Regulatory domain** → Controlled proteolysis

## Evolutionary Innovation
Domain shuffling created **vertebrate-specific innovations**:
- **Cartilage matrix proteins** (new domain combinations)
- **Inner ear proteins** (unique vertebrate architecture)
- **Complement cascade** (immune defense)
- ~1000 new domain pairs arose in vertebrate evolution

## Combinatorial Explosion
Limited domain inventory (10⁴) → vast proteome diversity (10⁶⁺):
- **Combinatorial**: 10,000 domains in sets of 2-5 → enormous space
- **Permutational**: Domain order matters (A-B ≠ B-A)
- **Stoichiometric**: Variable domain copy number (A, AA, AAA)

## Connection to Equidecomposability
Domain shuffling is **literal equidecomposability** in protein evolution:
- Proteins are "cut" at domain boundaries
- Domains are "pieces" that rearrange
- Same pieces, new arrangements → new functions
- Conservation of domain structure + variation in architecture

## Distinction from Moonlighting
- **Domain shuffling**: Physical rearrangement of modular units (evolutionary timescale)
- **Moonlighting**: Same structure, different contextual uses (no rearrangement)

Both involve reusing existing components for new functions.
